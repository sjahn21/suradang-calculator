## 🎯 실제 예시: 수라당 파렛트 분배 계산기

### 1. 프로젝트 개요

**프로젝트명**: 수라당 파렛트 분배 계산기

**한 줄 설명**: 떡 생산량을 파렛트에 최적으로 분배하여 줄수와 판수를 자동 계산하는 웹앱

**배경 및 목적**:
- 수라당에서 떡을 생산한 후 파렛트에 담아 쌓을 때 매번 계산이 필요
- 손으로 계산하면 시간이 걸리고 실수 가능
- 20~23개씩 쌓는 것이 최적이지만 매번 계산 필요

**주요 사용자**:
- 수라당 생산 관리자
- 물류 담당자
- 현장 작업자

**핵심 가치**:
- 계산 시간 단축 (5분 → 10초)
- 계산 오류 제거
- 최적 분배로 공간 효율 향상

---

### 2. 문제 정의

**현재 상황**:
- 계산기나 엑셀로 수동 계산
- 여러 경우의 수를 일일이 테스트
- 최적 분배를 찾기 어려움

**불편한 점**:
- 시간 소요 (매번 5분 이상)
- 계산 실수 발생
- 현장에서 즉시 확인 어려움 (PC 필요)
- 다양한 분배 방식 비교 어려움

**기대 효과**:
- 계산 시간 95% 단축
- 모바일에서 즉시 확인 가능
- 최적 분배 자동 제안
- 다양한 분배 방식 즉시 비교

---

### 3. 핵심 기능 (MVP)

**필수 기능**:
✅ 총 생산량 입력
✅ 파렛트당 개수 입력 (기본값 100)
✅ 3가지 분배 방식:
   - 스마트 분배 (20-23 자동)
   - 줄 수 우선
   - 파렛트 수 우선
✅ 결과 계산 및 표시
✅ 리셋 기능

**선택 기능** (v1.1):
- 계산 이력 저장
- 즐겨찾기 설정
- 결과 공유 기능
- 다크 모드

**제외 기능** (v2.0):
- 데이터베이스 연동
- 사용자 계정
- 통계 및 리포트
- API 제공

### 4. 사용자 시나리오

**시나리오 1: 일반적인 사용**
```
1. 작업자가 현장에서 스마트폰으로 접속
2. 오늘 생산량 12000개 입력
3. 파렛트당 100개 (기본값 유지)
4. 스마트 분배 선택 (기본값)
5. [계산하기] 버튼 클릭
6. 결과 확인: "총 6줄 + 0판, 한 줄당 20판"
7. 결과대로 파렛트 쌓기 시작
```

**시나리오 2: 특정 줄 수로 쌓고 싶을 때**
```
1. 창고 공간이 6줄만 가능한 상황
2. "줄 수 우선" 선택
3. 원하는 줄 수: 6줄 입력
4. 계산 결과: 한 줄당 20판 필요
5. 공간 확보 가능 여부 판단
```

**시나리오 3: 파렛트 수가 정해진 경우**
```
1. 파렛트가 25개씩만 들어가는 운송 차량
2. "파렛트 수 우선" 선택
3. 한 줄당 25판 입력
4. 계산 결과: 5줄 필요
5. 차량 적재 계획 수립
```

**시나리오 4: 에러 케이스**
```
1. 총 수량 입력 안 함
2. [계산하기] 클릭
3. 에러 메시지: "모든 항목을 입력해주세요"
4. 빨간색 에러 박스로 명확하게 표시
```

### 6. 데이터 구조

**입력 데이터**:
```javascript
{
  totalCakes: 12000,        // 총 생산량
  perBox: 100,              // 파렛트당 개수
  mode: "smart",            // "smart" | "rowPriority" | "palletPriority"
  desiredRows: 6,           // 줄 수 우선일 때
  boxesPerRow: 20           // 파렛트 수 우선일 때
}
```

**출력 데이터**:
```javascript
{
  mode: "스마트 분배",
  boxesPerRow: 20,          // 한 줄당 판수
  totalRows: 6,             // 전체 줄수
  fullRows: 6,              // 완전히 채운 줄
  lastRowBoxes: 0,          // 마지막 줄 판수
  lastBoxCakes: 100         // 마지막 판 떡 개수
}
```

**계산 로직**:
```javascript
// 1. 총 파렛트 수 계산
totalBoxes = Math.ceil(totalCakes / perBox)

// 2. 모드별 계산
if (mode === "smart") {
  // 20-23 중 최적값 찾기
  for (i = 20; i <= 23; i++) {
    score = 마지막줄이_가장_많이_채워지는_점수
    if (score < bestScore) {
      bestBoxesPerRow = i
    }
  }
}
else if (mode === "rowPriority") {
  boxesPerRow = Math.ceil(totalBoxes / desiredRows)
}
else if (mode === "palletPriority") {
  boxesPerRow = 사용자입력값
}

// 3. 최종 계산
fullRows = Math.floor(totalBoxes / boxesPerRow)
lastRowBoxes = totalBoxes % boxesPerRow
```

## 💡 핵심 정리

### 좋은 기획서는:
1. **명확하다**: 모호한 표현 없음
2. **구체적이다**: 숫자와 예시 포함
3. **실행 가능하다**: 기술적 제약 고려
4. **우선순위가 있다**: 필수 vs 선택
5. **검증 가능하다**: 성공 지표 명시

### 기획서 없이 시작하면:
- ❌ 개발 방향 자주 변경
- ❌ 반복 수정 증가
- ❌ 기능 누락 발생
- ❌ 일관성 없는 UI/UX

### 기획서를 만들면:
- ✅ 명확한 개발 방향
- ✅ 효율적인 프롬프트 작성
- ✅ 빠른 의사결정
- ✅ 일관된 결과물
